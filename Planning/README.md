# Planning

Strategy, plan, naming, and roadmap for Instar. Non-code artifacts that guide *what* we build and *why*.

## Contents

- **`Project-Plan.md`** — the plan. IP boundary, license reasoning, repo structure, versioning, contribution model, docs surface, CI, security policy, two-week sprint, first-90-days, success metrics, open questions.
- **`Naming.md`** — why "Instar," what off-box verifications (PyPI, domains, trademark) are still owed before public announcement.

## Not in this directory

- Governance/tooling files (`CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md`, `CHANGELOG.md`, `pyproject.toml`, `.github/`) live at repo root — GitHub and Python tooling expect them there. See `../CLAUDE.md` §Organization principle.
- Engineering notes, design decisions, and code live under `../Engineering/`.
- Positioning, announcements, and external communications live under `../Marketing/`.

## Adding new planning artifacts

Keep them focused. If a doc grows past ~20 KB or covers more than one concern, split it. Prefer editing the plan over adding a new doc — the plan is a living document.
